## 2. 用户原始目标

使用一个 2D 雷达，对一组：

- 高度相同；
- 相邻间距相等；
- 外形为矛头或尖头；

的目标进行检测，并输出每一个矛头各自的独立位置。

最终系统不能只输出整排目标的中心或一条拟合直线，而应为每个矛头分配稳定、可区分的编号，并输出各自的位置坐标。

### 2.1 已知目标结构

用户提供的结构图显示：

- 目标位于同一个固定矛头架上；
- 一共有 6 个安装槽位；
- 6 个矛头沿一条直线排列；
- 各矛头外形、横向宽度和朝向并不完全相同；
- 两端为较长的尖头形目标，中间包含多种较宽或较窄的异形目标；
- 项目重点应理解为定位 6 个独立安装槽位/矛头参考轴，而不是假设 6 个目标具有完全相同的扫描轮廓。
- 六个矛头按中轴线镜像对称排列；从一端到另一端的类型序列为 `牙尖、拳、掌、掌、拳、牙尖`，即左右对称位置属于同一类型。
- 每个矛头通过快速接头安装在直径约 6 mm、高约 28 mm 的小圆柱上。
- 用户已确认摆放方法：目标物块（当前圆柱）的中心直接对准预先确定的目标点。
  两条切线交点、圆柱边缘、近侧表面回波和矛尖都不是本轮人工真值点。

这意味着不能直接把每个点簇的几何中心都当成统一意义下的“矛尖位置”。不同外形和朝向会造成点簇中心偏移。优先方案是让雷达扫描平面穿过 6 个目标形状较一致的安装杆、颈部或安装基准截面，再利用已知共线、固定数量和近似等间距约束恢复 6 个槽位中心。如果必须扫描异形矛头主体，则需要为 6 个槽位分别建立模板或偏移标定。

对于沿自身中轴左右对称的目标，改变扫描高度 Z 后，截面在排列方向上的横向中心理论上仍接近安装轴线；但 2D 雷达只能看到朝向雷达的近侧轮廓，点簇的径向中心可能随截面深度变化。因此不能假设完整二维点簇质心在不同 Z 下严格不变。后续应优先以横向对称轴、安装小圆柱圆心或经过模板补偿的参考轴作为位置。

识别可利用固定结构先验：

1. 候选目标近似共线、数量为 6、间距近似固定；
2. 沿排列轴排序后，类别序列固定为 `牙尖、拳、掌、掌、拳、牙尖`；
3. 左右镜像槽位应具有相似的截面宽度、点数和轮廓特征；
4. 若装配顺序永久固定，可先识别 6 个槽位，再按槽位直接赋类型，不必完全依赖单帧轮廓分类；
5. 若顺序可能变化，则单层固定高度只有在三类目标截面可区分时才能分类，否则需要多高度扫描或额外传感器。

## 3. 当前对需求的工程化解释

建议将任务定义为：

1. 从 2D 激光雷达的一帧极坐标扫描数据中提取有效点。
2. 将雷达数据转换到笛卡尔坐标系。
3. 去除背景、墙面、地面边缘、离群点和超出工作区域的点。
4. 将属于不同矛头的点分成独立点簇。
5. 估计每个矛头在扫描平面中的代表位置。
6. 利用“矛头近似共线、间距近似相等”的先验条件，补强识别、排序和误检剔除。
7. 按固定方向排序，为每个矛头分配独立 ID。
8. 如需连续运行，进行跨帧匹配，使 ID 不轻易跳变。
9. 输出每个矛头在指定坐标系中的位置、置信度和状态。

### 3.1 “独立位置”的当前数据定义

当前已冻结的几何定义是：

```text
目标真实位置 = 目标物块/圆柱底面中心在扫描平面中的 X、Y
```

雷达直接看到的是目标表面回波，因此算法需要从表面点簇估计并补偿到上述中心；
不能把最近表面点或点簇边缘直接当作最终坐标。

每个检测结果至少包含：

```text
id              矛头编号，例如 0、1、2……
x               目标在输出坐标系中的 X 坐标，单位 m
y               目标在输出坐标系中的 Y 坐标，单位 m
range           到雷达原点的距离，单位 m
bearing         相对雷达坐标系的方位角，单位 rad 或 degree
confidence      检测置信度，范围 0～1
observed        当前帧是否直接观测到
timestamp       雷达帧时间戳
```

如果最终需要机器人或其他设备使用结果，还应明确输出的是：

- 雷达坐标系 `lidar_frame`；
- 设备/机器人基座坐标系 `base_frame`；
- 世界或工位坐标系 `world_frame`。

默认情况下，在没有外参标定数据之前，只能可靠输出雷达坐标系中的二维位置。

## 4. 必须牢记的传感器物理限制

单层 2D 激光雷达只测量一个扫描平面中的距离和角度，不能仅靠单帧数据测量目标的三维高度。

因此，“等高”目前按以下方式理解：

- 所有矛头与同一个雷达扫描平面相交；
- 2D 雷达看到的是各矛头在该平面中的截面或轮廓；
- 算法只估计该扫描平面内的二维位置 `(x, y)`。

如果用户实际上需要每个矛头的三维位置 `(x, y, z)`，则必须增加至少一种能力：

- 已知且固定的矛头高度，将 `z` 作为配置常量；
- 让 2D 雷达沿竖直方向移动并融合多层扫描；
- 使用多线激光雷达、深度相机或其他 3D 传感器；
- 通过机械结构和标定关系推算高度。

在用户确认之前，不应宣称系统能够由单层 2D 雷达直接测出高度。

#### 工作原理与结构

- 雷达类型：旋转式单层 2D DTOF 激光雷达
- 测距原理：发射红外激光，通过发射与接收的时间差计算光飞行时间，再换算距离
- 点云形成：距离数据与角度测量单元的角度值融合，形成二维点云
- 内部结构：激光测距核心、无线传电单元、无线通信单元、角度测量单元、电机驱动单元和机械外壳
- 外部通信接口：UART，波特率 `921600`
- 外形与装配孔：与 LD19 雷达相同

#### 暂存疑点

- 用户提供的产品特点写“满足 60Klux 环境使用”，参数表写“抗环境光 60Lux”。两者相差 1000 倍，且后者可能是漏写 `K`。在取得官方规格书前，本文不选择其中任何一个作为确定值。
- “无线通信单元”描述的是雷达内部旋转部分与外部接口之间的数据传输，不应误解为雷达对上位机提供 Wi-Fi 或蓝牙；当前已知的外部接口仍是 UART。
- 已知 UART 波特率，但数据帧格式、校验方式、角度编码、距离单位、强度字段和转速字段仍未知，尚不能编写可靠的数据解析器。
- “俯仰角误差 0.5°～2°”可能显著影响细小矛头能否稳定落在扫描平面内，需要结合安装高度、目标距离和矛头有效高度进行机械容差分析。

#### USB 转串口转接板选型

2026-06-18 根据厂商工具和驱动检查：

- 厂商 Windows 查看器明确提示使用 CP2102。
- 官方 udev 规则匹配 Silicon Labs CP210x 的 USB VID:PID `10c4:ea60`。
- STL-27L 使用 UART `921600`，应购买支持该波特率的 TTL UART 转接板。

购买优先级：

1. 向 STL-27L/LDROBOT 卖家购买明确标注兼容 STL-27L 的原厂 USB 转接板和原装线束，并让卖家确认 Linux、921600 baud、5V 供电和插头线序。
2. 通用方案优先选择 CP2102N，例如 `Adafruit CP2102N Friend, Product ID 5335`。该板支持最高 3 Mbps，提供 5V 电源线且 UART 信号为 3.3V。
3. 可选 `Waveshare CP2102 USB UART Board (Type C), SKU 20644`。其官方规格为 TTL UART、300 bps～1 Mbps、Linux 支持、3.3V 逻辑兼容和 5V/3.3V 输出，因此理论上覆盖 921600 baud。

禁止或不推荐：

- 不要购买 USB-RS232、USB-RS485 或带 DB9 的传统串口板；它们不是 TTL UART。
- 不要在未确认接口电平前，把 5V UART 信号直接接入雷达数据脚。
- 不要仅按导线颜色判断 5V、GND、TX、RX/PWM。
- 廉价 CH340/无品牌 CP2102 板可能在 921600 下不稳定，只适合备选。

STL-27L 标称工作电流 ≤290 mA。通用转接板若通过 USB 5V 给雷达供电，应确认 5V 引脚是 USB VBUS 直通且板、线和 USB 口可稳定提供至少 500 mA；更稳妥的实验方案是单独使用稳定 5V/1A 电源给雷达供电，并与 USB-UART 共地。接线前仍需从卖家手册取得 STL-27L 插座针脚定义。

#### 基于规格的理论推导

以下是推导值，不是实测结果：

- 默认 10 Hz 时，理论测距次数约为 `21600 / 10 = 2160` 次/圈。
- 如果这些测距在 360° 内近似均匀分布，则理论平均角间隔约为 `360 / 2160 = 0.167°`。
- 在 6 Hz 时约为 3600 次/圈、0.100°；在 13 Hz 时约为 1662 次/圈、0.217°。
- 目标在距离 `R` 处，相邻光束的横向间距可近似为 `R × Δθ`。默认 10 Hz 下约为：1 m 处 2.9 mm、2 m 处 5.8 mm、5 m 处 14.5 mm。

以上估算假设测距均匀、每次测距都形成有效点，实际有效点数还会受到数据协议、无效回波、目标反射率、转速波动和固件处理影响。分簇阈值不能仅根据这些理论值固定。

## 7. 推荐算法方案

### 7.1 输入标准化

原始雷达帧通常为：

```text
[(angle_0, range_0, intensity_0), ...]
```

笛卡尔坐标转换：

```text
x = range * cos(angle)
y = range * sin(angle)
```

需要保留原始角度、距离、强度和时间戳，不能只保留 `(x, y)`，否则后续难以排查驱动和传感器质量问题。

### 7.2 预处理

推荐顺序：

1. 删除无效距离、零值、NaN 和超出雷达有效量程的点。
2. 使用工作区域 ROI，只保留矛头可能出现的范围。
3. 如有固定墙面或支架，使用静态背景模型去除。
4. 使用中值滤波、邻域距离过滤或统计离群点过滤抑制飞点。
5. 如果目标反射强度有明显特点，可将强度作为辅助条件，但不能在未验证前硬编码。

### 7.3 候选点分簇

第一版建议采用基于相邻扫描点的自适应分割，而不是直接使用固定欧氏阈值：

- 按雷达角度顺序遍历点；
- 比较相邻点的欧氏距离；
- 阈值随测量距离和角分辨率变化；
- 距离突变时切分点簇；
- 对点数、宽度、深度不符合矛头截面范围的点簇进行过滤。

也可以用 DBSCAN 作为基线实现，但 `eps` 必须考虑检测距离变化。远距离相邻激光束之间的横向间隔更大，固定 `eps` 可能导致近处误合并、远处误拆分。

### 7.4 单个目标的位置估计

用户已确认最终目标点为目标物块/圆柱中心。以下方法是从雷达表面回波恢复该中心的候选方法：

1. 点簇质心：稳定、简单，但不一定对应尖端。
2. 距雷达最近点：可能接近突出尖端，但容易受噪声影响。
3. 沿矛头主轴方向最突出的点：更符合“矛尖”，但需要知道轮廓方向。
4. 对矛头两侧边缘拟合直线，取交点作为尖端：几何意义最好，但要求每个矛头有足够扫描点。
5. 使用已知模板匹配中心或尖端：适合尺寸固定、数据质量足够的场景。

调试阶段可同时输出：

- `cluster_center`：点簇稳健中心；
- `target_center`：经过半径、轮廓或模板补偿后的目标中心；
- `position_method`：本次结果使用的方法。

最终误差统一比较 `target_center` 与用户摆放时的中心目标点。当前代码仍主要输出稳定点簇中心，
下一步需要利用多组真值估计“表面回波中心→物块中心”的系统偏移。

### 7.5 共线和等间距约束

等间距是本项目最重要的结构先验，建议用于候选验证，而不是替代真实观测。

基本步骤：

1. 对候选中心使用 RANSAC 或稳健最小二乘拟合排列轴线。
2. 将每个候选位置投影到轴线，得到一维坐标 `s_i`。
3. 按 `s_i` 排序。
4. 计算相邻差值 `d_i = s_(i+1) - s_i`。
5. 用中位数或已知设计值估计标准间距 `pitch`。
6. 检查每个候选是否接近某个整数槽位：

```text
s_k ≈ s_0 + k * pitch
```

7. 剔除明显不共线或不满足间距规律的误检。
8. 对短时漏检的槽位标为 `predicted/missing`，不得伪装成直接观测结果。

如果已知矛头总数 `N` 和标准间距，算法会明显更稳定；如果两者未知，则需要从数据估计，容易受到遮挡和误检影响。

### 7.6 编号规则

独立 ID 必须有明确、可重复的方向。建议：

- 将所有目标投影到排列轴；
- 规定排列轴正方向；
- 沿正方向从 `0` 到 `N-1` 编号。

排列轴方向可通过以下方式固定：

- 在工位坐标系中规定从左到右；
- 使用已标定的起始端参考物；
- 使用距离某个固定基准点最近的目标作为 `id=0`；
- 对连续帧使用上一帧方向和 ID 约束，防止整排 ID 反转。

只按每帧临时拟合直线排序，可能出现整排编号突然反转，必须专门处理。

### 7.7 连续帧跟踪

如果目标或雷达会运动，建议加入：

- 匀速卡尔曼滤波或指数平滑；
- 最近邻、匈牙利算法或槽位索引的数据关联；
- 最大允许位移门限；
- 短时漏检保留；
- 新建、确认、丢失、删除等轨迹状态。

如果矛头和雷达完全固定，可先不做复杂跟踪，改用多帧中位数降低噪声。

### 7.8 坐标标定

雷达坐标到工位坐标的二维刚体变换：

```text
p_world = R(theta) * p_lidar + t
```

其中：

- `theta`：雷达坐标系与目标坐标系之间的旋转角；
- `t = [tx, ty]`：平移；
- 至少需要两个不重合的已知参考点，实际建议使用三个或更多点进行稳健拟合；
- 标定参数必须保存到配置文件，并记录单位和坐标轴方向。

## 11. 验收指标建议

最终验收不能只凭可视化“看起来正确”。建议至少统计：

- 检出率：真实矛头中被正确检测的比例；
- 误检率：输出中不对应真实矛头的比例；
- 数量正确率：每帧检测数量完全正确的比例；
- 单目标位置误差：与人工测量真值的欧氏距离；
- 95% 位置误差：避免平均值掩盖极端错误；
- 间距估计误差；
- ID 稳定性：连续帧中目标 ID 错换次数；
- 处理速度：平均耗时、95% 耗时和目标帧率；
- 遮挡恢复：目标重新出现后能否恢复原 ID。

用户当前允许考试结果约 `2 cm` 的位置误差。正式验收时应逐目标计算二维欧氏误差，
同时保留 `dx`、`dy` 和径向误差，不能只凭RViz视觉判断。

## 15. 风险与易错点

- 将“等高”误解为 2D 雷达能够测量高度。
- 将点簇质心直接称作“矛尖位置”，但二者可能有系统偏差。
- 使用固定聚类距离，导致不同测距下结果不稳定。
- 仅根据等间距补出目标，却没有标记该位置是推测值。
- 每帧独立排序导致 ID 整排反转或互换。
- 只在一组理想数据上调参，现场出现背景、遮挡后失效。
- 忽略雷达安装误差和坐标外参，导致局部检测准确但机器人坐标错误。
- 角分辨率不足时，单个矛头只有极少点，复杂轮廓拟合没有意义。
- 仅用 `21600 Hz / 扫描频率` 推算每圈点数可能高估有效点数，必须以实际数据帧为准。
- STL-27L 的俯仰角误差和安装倾斜可能使扫描平面错过细小矛头，不能只靠软件补偿。
- 不同反射率下的有效量程差异明显；矛头材质和表面角度可能造成丢点。
- 矛头间距在角度域不相等；必须先转为笛卡尔坐标或投影到排列轴后比较实际距离。
- 把雷达标称精度等同于最终定位精度；最终误差还包含采样、安装、标定、目标表面和算法误差。
