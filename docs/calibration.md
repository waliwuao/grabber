## 外壳距离换算

用户确认量尺起点是下层方形底座表壳，而不是上方圆形扫描头。
下层底座主体约为 `38.5 × 38.5 mm`，因此采用：

- 旋转中心到下层底座表壳：`38.5 / 2 = 19.25 mm`

因此四个方向的旋转中心到物体表面标定真值相同：

| 方向 | 外壳100 mm | 外壳200 mm |
|---|---:|---:|
| `+X/-Y/-X/+Y` | 119.25 mm | 219.25 mm |

`54.00 × 46.29 mm` 是包含安装耳的底座外廓；`46.80 mm`
和 `31.92 mm` 是安装孔定位尺寸，不能直接除以2作为表壳补偿。
本换算也不使用突出的安装耳；量尺应接触方形底座主体的平面外壁。

## 距离修正方程

对雷达原始点 `(r_raw, theta)` 使用：

```text
r = 0.96370434*r_raw
    - 0.00795114
    - 0.00072278*cos(theta)
    - 0.00192741*sin(theta)
    - 0.00036139*cos(2*theta)
```

单位均为米，`theta` 使用雷达消息中的角度。修正后的二维点为：

```text
x = r*cos(theta)
y = r*sin(theta)
```

四个标定方向可简写为：

```text
+X: r = 0.96370434*r_raw - 0.00903530
-Y: r = 0.96370434*r_raw - 0.00566234
-X: r = 0.96370434*r_raw - 0.00758975
+Y: r = 0.96370434*r_raw - 0.00951716
```

扫描平面在雷达自身坐标系中的表达式仍为：

```text
z = 0
```

## 标定拟合检查

八个标定位置的内部拟合结果：

```text
RMS残差约 0.63 mm
最大残差约 0.86 mm
```

这只是拟合残差，不代表绝对真值精度。考虑人工测距约±2 mm、外壳尺寸取值、标定物摆放和雷达标称误差，盲测前合理的绝对不确定度预计仍为数毫米，不能把0.63 mm宣称为最终定位精度。

角度未使用这批数据校正，因为四个方向只是人工粗略摆放；用它校正角度会把摆放误差写入雷达参数。
## 六目标物块中心补偿 V1（2026-06-19）

径向标定公式继续保留，不被本次拟合替换。雷达点经过径向标定、多帧网格和
稳定区域识别后，得到的是物体表面回波区域中心；用户定义的真值则是物块/圆柱中心。
因此在识别后增加第二层中心补偿。

拟合数据：

```text
data/bags/six_targets_0mm_r1_20260619_171750/bag
```

目标按识别 ID 从 `-X` 到 `+X` 排列。在目标排列线局部坐标系中：

```text
p_center[i] = p_component[i]
              + along_offset[i] * row_axis
              + normal_offset[i] * row_normal
```

V1 参数（单位 m）：

```text
along  = [0.02003030, 0.00502586, 0.01283923,
          0.01502585, 0.02503029, 0.04003027]
normal = [0.01696429, 0.01449106, 0.02197713,
          0.01447321, 0.01695537, 0.01692860]
```

### 当前启用状态（2026-06-19）

独立数据验证表明，V1 的逐ID固定 `along` 补偿会随目标距离和观察角度变化而失效，
并将部分原本较小的X误差放大到约43 mm。因此当前运行参数已改为：

```text
along  = [0, 0, 0, 0, 0, 0]
normal = [0.01696429, 0.01449106, 0.02197713,
          0.01447321, 0.01695537, 0.01692860]
```

旧V1 `along` 数值仅作为历史拟合结果保留，不再用于在线或离线输出。后续需要使用
多组不同距离和角度的数据，将沿排列方向补偿建模为 `r/θ` 的函数。

## 六目标极坐标补偿 V2（2026-06-19）

V2仅使用以下三组有效六目标数据及表格真值：

```text
six_targets_0mm_r1_20260619_171750
six_targets_0mm_r1_20260619_175815
six_targets_0mm_r1_20260619_185117
```

`six_targets_0mm_r1_20260619_183046` 的 `scene.md` 记录为4个目标，且该组误差较大、
没有写入真值表，因此明确排除，不参与拟合或验收。

在保留法向中心补偿后，令：

```text
u = (r - 0.20) / 0.15
v = (theta + pi/2) / (pi/2)
f = [1, v, u, v², u*v, u²]

r_corrected     = r     + dot(Cr, f)
theta_corrected = theta + dot(Ctheta, f)
```

系数为：

```text
Cr = [ 0.00883056907398,  0.00494075146535, -0.00461366266983,
      -0.0000114361849171, 0.03486982091,   -0.0024647133478 ]

Ctheta = [ 0.0898957868697,  0.0668936983864,  0.037292691619,
          -0.115131471909,  -0.108171945595,  -0.0783418855768 ]
```

单位分别为米和弧度。18个参与拟合点上的结果为：

```text
X平均绝对误差：约14.4 mm
X最大绝对误差：约33.5 mm
Y平均绝对误差：约3.6 mm
Y最大绝对误差：约8.6 mm
```

这是标定内结果，不是独立精度。整包留出验证显示X误差改善有限，说明剩余X误差主要
来自不同距离下可见表面区域变化，不能把V2解释为已经完成精密雷达角度标定。必须用
下一组未参与拟合的六目标数据冻结验证，不能继续提高多项式阶数把当前18点拟合到零。

### V2独立验证结论：未通过

冻结参数后使用第四组独立数据：

```text
six_targets_0mm_r1_20260619_192111
```

其真值位于 `data_contract.ods` 第12行。对比结果：

| 模型 | X平均绝对误差 | Y平均绝对误差 | 平均二维误差 |
|---|---:|---:|---:|
| 仅保留法向补偿 | 11.6 mm | 9.9 mm | 17.2 mm |
| 法向补偿 + V2 | 8.9 mm | 15.7 mm | 19.9 mm |

若以X、Y和总体二维误差共同验收，V2未通过。但用户于2026-06-20明确当前阶段只优先
保证X精度，Y暂不参与验收。按这一目标，V2使第四组X平均绝对误差从11.6 mm降至
8.9 mm，并将X误差不超过10 mm的目标数从3/6提高到5/6，因此在线和离线流程已重新
启用V2。该决策不代表Y或二维总体精度已经达标。

采用排列线局部坐标而非固定全局 X/Y，使整排目标轻微旋转时补偿方向也随之旋转。
输出 JSON 同时保留：

```text
raw_component_x_m / raw_component_y_m：补偿前表面区域中心
x_m / y_m：补偿后的物块中心
```

本组数据参与了参数拟合，因此训练残差为零不代表独立精度。下一包改变位置的数据
必须在不修改 V1 参数的情况下验证；只有独立数据的 `|dx|`、`|dy|` 均不超过5 mm，
才能认为达到当前精度目标。
