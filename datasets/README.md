## 目标

- 有效工作距离：目标真值距离不超过 0.40 m。
- 目标定位误差要求：厘米级，目标值为绝对误差不超过 10 mm。
- 处理 ROI 暂设到 0.43 m，额外 3 cm 只用于容纳测距正误差，不代表允许目标超过 0.40 m。

## 精度现实约束

STL-27L 在 0.03～2 m 的用户提供标称参数为均值误差 ±15 mm、标准差 5 mm。因此：

- 单个原始点或单帧绝对距离不能预先保证 ≤10 mm。
- 静止目标可通过多帧中位数、系统偏差标定和结构先验争取达到厘米级。
- 最终是否满足 ≤10 mm，必须用独立真值测量和未参与标定的测试场景验证。
- 真值工具误差应明显小于 10 mm，建议不大于 2～3 mm。

## 真值距离定义

用户已确认采集距离定义为：

```text
雷达旋转中心在扫描平面内 → 最近矛头参考点的水平直线距离
```

当前安装方向更新为从绿色负轴 `-Y` 一侧测量。雷达不要求放在两个目标的正中间。两个目标沿红色 `X` 轴排列时，建议记录三个真值：

```text
d：雷达旋转中心到目标排列线的垂直距离
p：两个目标参考轴之间的实际间距
a：雷达垂足到 ID0 的有符号 X 方向偏移
```

则两个目标的理论雷达坐标为：

```text
ID0 = (a,     -d)
ID1 = (a + p, -d)
```

其中 `a` 可为正或负。只要两个目标斜距均不超过 400 mm，雷达可以偏置放置。

当前参考点已由用户固定为：

```text
预先确定平面目标点，并将目标物块/圆柱底面中心直接放在该目标点上。
```

所有场景必须使用这个中心定义。圆柱边缘、两条切线的交点、近侧表面回波和矛尖
均不作为本轮真值。若目标是获得每个矛头的独立二维位置，还必须记录：

- 相邻矛头的实际中心间距；
- 矛头数量；
- 整排矛头相对雷达的排列方向；
- `id=0` 所对应的固定一端。

根据用户提供的结构图，架上固定有 6 个外形和朝向不同的目标。当前真值参考点统一定义为：

```text
每个目标物块/圆柱中心与雷达扫描平面的交点
```

不要对 6 种不同外形分别使用各自轮廓中心，否则得到的坐标不具备统一机械意义。

雷达安装高度优先选择扫描 6 个目标形状一致的安装杆、颈部或底座截面。如果只能扫描异形主体，应为 6 个槽位分别测量“回波中心到安装轴线”的偏移量。

不要只测雷达外壳到目标表面的距离；雷达旋转中心与外壳边缘存在固定偏移，需要测量或标定。

## 第一轮必录数据

当前只有两个真实矛头。第一轮优先使用这两个真实目标，不使用其他材质凑数。采用 150、200、250、300、350、400 mm 六个距离，每个静态场景录制 30 秒，并独立重复 3 次：

| 场景 | 真值距离 |
|---|---:|
| 空背景 | 0 mm 占位 |
| 两个真实矛头 | 150 mm |
| 两个真实矛头 | 200 mm |
| 两个真实矛头 | 250 mm |
| 两个真实矛头 | 300 mm |
| 两个真实矛头 | 350 mm |
| 两个真实矛头 | 400 mm |

共计：

```text
6 个距离 × 3 组 = 18 组双矛头数据
另加至少 1 组空背景数据
```

每一组开始前应停止录制，再重新开始一份 rosbag。三组重复数据之间，至少等待数秒；更严格的测试可以轻微断电重启雷达或重新启动驱动，以覆盖重复启动误差。

此外录制：

- 阵列左转约 10°、20°；
- 阵列右转约 10°、20°；
- 遮挡一个矛头；
- 阵列附近放置一个干扰物。

## 现场布置

1. 固定雷达，不能手持。
2. 固定矛头阵列，录制过程中不能移动。
3. 扫描平面必须穿过所有矛头的同一高度。
4. 用钢直尺、卡尺或可靠激光测距设备记录真值。
5. 测量每个矛头之间的实际间距，而不只记录设计值。
6. 记录雷达旋转中心相对外壳的参考位置。
7. 保持电源、转接板和线缆位置不遮挡扫描区域。
8. 记录扫描平面穿过目标的具体高度，并拍摄侧视照片。
9. 从固定端开始将 6 个槽位编号为 0～5，之后所有采集保持编号方向不变。
10. 当前双目标测试从 `-X` 到 `+X` 编号，即较小 X 坐标为 ID0；若现场固定端相反，可将配置 `id_sort_ascending` 改为 `false`。


- 录制后显示 rosbag 信息。

## 录制后一条命令处理并查看

处理最新录制的bag（默认固定6个目标）：

```bash
cd /home/charlie/2d雷达
./scripts/process_and_view_bag.sh
```

也可以明确指定场景目录：

```bash
./scripts/process_and_view_bag.sh \
  data/bags/background_empty_0mm_r1_20260619_143330
```

或直接指定其中的bag目录：

```bash
./scripts/process_and_view_bag.sh \
  data/bags/background_empty_0mm_r1_20260619_143330/bag
```

该命令会依次：

1. 离线处理完整rosbag；
2. 将结果保存为场景目录下的 `recognition_result.json`；
3. 启动识别节点；
4. 循环回放该rosbag；
5. 打开RViz显示原始点云、六个目标圆圈、ID和 `(x,y)`。

按一次 `Ctrl+C` 会停止回放、识别节点和RViz。
