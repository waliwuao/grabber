## 当前任务

暂缓绝对位置精度标定，先识别二维扫描中哪些稳定区域属于目标。

数据标签解释已修正：

- `_r5_` 表示第5次重复录制，不表示两个目标；
- 目标数量必须读取 `scene.md` 中的“矛头数量”，不能从目录后缀猜测；
- `background_empty` 表示无目标。

## 方法

单帧点簇在真实数据中容易被无效回波切成2～3段，因此采用：

1. 仅保留绿色负轴 `-Y` 一侧、0.08～0.43 m ROI；
2. 将多帧点投票到5 mm二维网格；
3. 保留在至少15%帧中出现的稳定网格；
4. 合并相邻稳定网格为候选区域；
5. 排除跨度过大的墙面和环境结构；
6. 按期望数量选择紧凑、共线的候选目标；
7. 从候选中心自动拟合排列轴，不要求排列轴必须等于红色X轴；
8. 沿自动拟合的排列轴排序编号。

当前 `-Y` 角度和矩形ROI只是可配置的搜索窗口，用于排除电脑、桌腿和墙面，不参与目标类别判断。测试全360°搜索时，200 mm数据会被房间内其他紧凑物体干扰，因此现阶段不应取消ROI，而应在实际安装后将ROI设置为“架子可能出现的区域”。完整6目标时，共线和等间距约束会比双目标强得多。

完整6目标时将利用：

- 固定数量6；
- 近似共线；
- 近似等间距：任一相邻间距相对平均间距的偏差默认不超过35%；
- 类型顺序 `牙尖、拳、掌、掌、拳、牙尖`。

固定数量并不是问题，最终在线配置默认就是6。50 mm双目标bag属于算法调试数据，回放时临时使用 `expected_count:=2`。

## 排列线与扫描平面

当前算法从目标中心拟合的是二维排列直线，不是三维平面。输出形式为：

```text
a*x + b*y + c = 0
```

其中 `(a,b)` 是单位法向量。若雷达坐标系定义为扫描平面就是 `XY` 平面，则三维扫描平面表达式是 `z = 0`。变换到其他坐标系后，若外参为旋转 `R`、平移 `t`，平面法向量为 `n = R*[0,0,1]^T`，平面为：

```text
n_x*x + n_y*y + n_z*z + d = 0
d = -n·t
```

单层2D雷达数据本身不能独立估计扫描平面在三维空间中的姿态；必须使用安装外参或额外标定数据。

```bash

```
